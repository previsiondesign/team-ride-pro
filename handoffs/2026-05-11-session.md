## Strava proxy — design rationale
Strava's main route pages (`strava.com/routes/{id}`) now require login. Unauthenticated fetches return "Log in to see '{name}'" with wrong distance/elevation. Fix: proxy (`server.js` on Render) tries `strava-embeds.com/route/{id}` first (public, no auth needed). That page has clean `.stat-label`/`.stat-value` HTML — parsed directly instead of regex on body text. Falls back to original scraping if embed fails. The proxy deploys to Render automatically from `main`.

## PDF nickname fix — why a new helper
`getDisplayName()` in app-utils.js ignores `nicknameMode`. The planner UI uses inline logic in `renderCoachCardHtml`/`renderRiderCardHtml` (app-groups.js) that checks `nicknameMode === 'firstName'` to replace only first name vs entire name. PDFs were using raw `.name`. Created `getPersonDisplayName(person, fallback)` that replicates the nicknameMode logic. Applied to all 8 locations: mobile PDF coaches (4), mobile PDF riders, desktop PDF coaches (4), desktop PDF riders, desktop height calc, print view riders, print view coaches.
